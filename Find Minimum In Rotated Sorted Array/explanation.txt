DEFAULT BINARY SEARCH ALGORITHM NOTES:
<= because need to check instance where low == high (one element in array).
    otherwise, the loop would not check subsection of array if, after moving left/right, only one element
    remains. If one element remains, low == high.
    
    example:
    [1,2,5]
    
    low = 0, mid = 1, high = 2 (indices), target = 5 (value)
    2 < 5, so low = mid+1. low = high = 2. 

    if we had low < high, this would cause the loop to break. we need one more pass to check the case
        where the target could possibly be the one remaining element.

mid = low + (high - low) // 2
        this mid prevents possible integer overflow if low+high exceeds the max integer size.
        high - low produces smaller number
        both versions are produce the midpoint, but commented alternative is safer

you subtract or add 1 to low / high when halving search space since midpoint is checked in conditionals

function logarithmically searches for value in monotonic/sorted array.
default binary search. need to modify algorithm for this problem.
not searching for specific value, but searching for minimum element

Because I am using a modified version of the binary search, my algorithm performance is O(logn).

def binarSearch(arr, target):
    low = 0
    high = len(arr) - 1
    
    while low <= high: 
        mid = (low + high) // 2
        if target < mid:
            high = mid - 1
        elif target > mid:
            low = mid + 1
        else:
            return mid
    return -1 # not a valid index
